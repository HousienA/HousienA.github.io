Patient Journal Microservices System

Full-stack, secure medical records platform built as five independently deployable services. A Spring Boot clinical backend handles patient diagnoses and records; a Kafka-backed messaging service powers secure patient–clinician communication; a Node.js image service manages file uploads via Multer; and a Quarkus reactive search service provides fast lookups across records. A React 19 frontend ties it together, with authentication delegated to Keycloak through OIDC. Each service is containerized with Docker and orchestrated on KTH Cloud using Kubernetes.

ATLA — Substation Monitoring System

Cloud-based monitoring system for simulated electrical substations, developed at KTH in partnership with Hitachi Energy. A Spring Boot simulator models AC devices (transformers, breakers, capacitor banks) and HVDC stations (monopolar and bipolar, with converters, valves, and reactors) and exposes live metrics in Prometheus format. Prometheus scrapes every 15 seconds, Grafana provides pre-provisioned dashboards, and a React frontend offers overview, per-device detail views, and historical charts. The whole stack runs under Docker Compose for one-command deployment.

Java Spring Boot Prometheus Grafana React Docker Compose

Cross-platform fitness app for iOS, Android, and web, built with React Native and Expo Router. Auction Web Service

Online auction platform built with ASP.NET Core MVC and Razor Pages on .NET. Uses Entity Framework Core with two separate MySQL database contexts — one for auction domain data, one for ASP.NET Core Identity — keeping authentication concerns cleanly isolated from the business schema. Layered architecture separates controllers from services from a persistence abstraction (IAuctionPersistence, IUserPersistence), with AutoMapper handling DTO-to-entity translation and Identity requiring confirmed-account sign-in.

N-Back Cognitive Training App

Android app implementing the N-back working-memory paradigm, where the player must identify whether the current stimulus matches the one shown N steps earlier. Built in Kotlin with Jetpack Compose and MVVM, with stimulus sequences generated in a native C library bridged through JNI — the Kotlin side calls out through a JniBridge shared library, and the C code handles sequence construction and match distribution. Supports visual and audio game modes (audio via SoundPool), with high scores persisted through Jetpack DataStore.

Kotlin Jetpack Compose C JNI DataStore

MonkeyShooter

Top-down arcade multiplayer shooter written from scratch in C with SDL2. A dedicated C server handles up to six simultaneous players over the network using SDL2_net, while the client renders sprites, text, and bullet physics through SDL2, SDL2_image, and SDL2_ttf. Shared game logic (characters, world, projectiles) lives in a common library used by both client and server, with build orchestrated through Makefiles targeting both Windows/MinGW and macOS.

C SDL2 Networking Multiplayer Make

Weather App Android

Native Android weather app in Kotlin with a clean MVVM architecture. Uses Retrofit to talk to the Open-Meteo forecast and geocoding APIs, mapping free-text place names to coordinates before fetching daily forecasts. Features include favorite locations, offline caching with a last-updated indicator, and graceful degradation across network-error, offline-with-cache, and offline-no-cache states. Organized as data/remote, data/local, data/repository, domain, and ui layers with a single source of truth in the repository.
